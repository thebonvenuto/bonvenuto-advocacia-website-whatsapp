# Bonvenuto Page Refinement

## Proposito

Refinar paginas existentes sem refazer o projeto inteiro. Melhorar clareza, hierarquia, ritmo e conversao mantendo o sistema visual atual.

## Quando usar

- Usar para melhorar hero, subtitulo, ordem de secoes e distribuicao de CTA.
- Usar para deixar uma pagina mais premium e mais clara sem trocar a identidade do site.
- Usar quando uma rota estiver correta em conteudo, mas fraca em leitura, densidade ou consistencia.
- Usar quando uma pagina nova precisar seguir o mesmo padrao visual da Bonvenuto.

## Quando nao usar

- Nao usar para redefinir o posicionamento inteiro do projeto.
- Nao usar para abrir novas areas juridicas ou mudar o objetivo principal de conversao.
- Nao usar como checklist final de QA ou revisao ampla do site inteiro.

## Instrucoes praticas

- Identificar primeiro a parte mais fraca da pagina: hero, sequencia, copy, CTA ou repeticao.
- Comecar pelo objetivo da rota: explicar o servico, gerar confianca e levar ao WhatsApp.
- Melhorar a pagina por blocos, nao por revolucao total.
- Garantir que o hero responda rapido: para quem e, qual dor organiza, qual proximo passo tomar.
- Manter um CTA principal por tela, quase sempre orientado a WhatsApp.
- Dar funcao clara para cada secao: explicar, gerar confianca, tratar objecao, orientar, converter.
- Reduzir excesso de texto antes de adicionar novas secoes.
- Preservar fontes, cores, botoes, contraste, linguagem e nivel de sobriedade do sistema atual.
- Evitar layouts genericos de cards sem funcao.
- Quando adicionar uma nova secao, fazer ela ganhar o direito de existir.

## Checklist rapido

- O hero ficou mais claro em menos tempo?
- A pagina tem ritmo melhor entre conteudo, respiro e CTA?
- O CTA principal aparece cedo e volta nos momentos certos?
- O caminho para o WhatsApp ficou mais claro sem poluir a pagina?
- A pagina parece parte do mesmo sistema visual?
- A mudanca refinou a rota sem descaracterizar o site?
