# Bonvenuto Mobile QA

## Proposito

Revisar paginas com foco real em mobile. Detectar problemas de leitura, hierarquia, densidade, empilhamento ruim e CTA fraco em telas pequenas.

## Quando usar

- Usar antes de considerar uma pagina pronta para publicacao.
- Usar quando o layout funciona no desktop, mas parece pesado no celular.
- Usar ao revisar hero, grids, cards, FAQs, imagens e sticky CTA.
- Usar sempre que uma rota nova ou refinada tocar estrutura visual.

## Quando nao usar

- Nao usar como substituto de revisao de copy.
- Nao usar para discutir escopo juridico ou produto.

## Instrucoes praticas

- Ler a pagina como um usuario em telefone, de cima para baixo, sem assumir contexto previo.
- Verificar se o hero cabe bem no mobile e se comunica a proposta sem parecer um desktop comprimido.
- Reduzir grids quebrados, colunas apertadas e excesso de cards empilhados.
- Checar legibilidade real: tamanho de fonte, entrelinha, comprimento de bloco e contraste.
- Garantir que o WhatsApp apareca cedo e continue facil de acionar.
- Verificar se o sticky CTA ajuda e nao cobre conteudo importante.
- Verificar se botoes, links e areas tocaveis sao confortaveis no dedo.
- Ajustar espacamento vertical para evitar secoes coladas ou longas demais.
- Priorizar fluxo simples: mensagem, prova, orientacao, CTA.
- Cortar decoracao que atrapalhe leitura ou gesto no mobile.

## Checklist rapido

- A pagina funciona bem em largura de telefone comum?
- O hero comunica sem rolagem cansativa?
- O texto continua legivel e respirado?
- O sticky CTA ajuda mais do que atrapalha?
- Os alvos de toque estao confortaveis?
- A rota parece desenhada para mobile, e nao apenas reduzida?
